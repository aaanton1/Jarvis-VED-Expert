## 1. Формула скоринга

```
overallScore = round((sumSuccessStrengths / (sumSuccessStrengths + sumDeathSeverities)) × 100)
```

**Пример:** strengths = 35, severities = 25 → 35/(35+25)×100 = **58/100**

Балл рассчитывается **серверно** из сырых данных LLM. Самооценка модели игнорируется (защита от галлюцинаций).

Код:
```typescript
const calculatedScore = total > 0
  ? Math.round((sumStrengths / total) * 100)
  : 50;
```

## 3. Структура оценки

### 3.1 Death Reasons (5 штук)

Каждый риск оценивается по `severity` (1–10):

| Severity | Описание                                                         |
|----------|------------------------------------------------------------------|
| 1–3      | Незначительный риск, легко митигируется                           |
| 4–6      | Умеренный риск, требует внимания и ресурсов                      |
| 7–8      | Серьёзный риск, может убить бизнес без правильной стратегии      |
| 9–10     | ТОЛЬКО регуляторный запрет или физическая невозможность           |

> Средняя severity для нормального B2B SaaS: **4–6**, не 8–10.

### 3.2 Success Drivers (5 штук)

Каждый драйвер оценивается по `strength` (1–10):

| Strength | Описание                                                         |
|----------|------------------------------------------------------------------|
| 1–3      | Слабое преимущество, легко копируется                            |
| 4–6      | Умеренное преимущество, даёт фору на 6–12 месяцев               |
| 7–8      | Сильное преимущество, создаёт устойчивый конкурентный барьер     |
| 9–10     | ТОЛЬКО уникальные, неповторимые преимущества                     |

## 4. Коэффициенты компенсации

### 4.1 Юнит-экономика (LTV/CAC)
Если LTV/CAC > 5, severity рисков внедрения **снижается на 2–3 пункта**.
Логика: прибыль оправдывает усилия по преодолению сложностей.

### 4.2 Стоимость ошибки
Если ошибка бота **НЕ** ведёт к смерти, потере здоровья или многомиллионным искам — severity юридических рисков **снижается на 2–3 пункта**.

### 4.3 Голубой океан
Если конкуренты не используют канал (Messenger Max) — это **сильный success driver**, а не «платформа-призрак». Max: 50M+ пользователей, Bot API бесплатный до 2027.

### 4.4 Масштабируемость
SaaS с AI масштабируется без найма штата — это драйвер, а не риск.

## 5. Интерфейсы данных

```typescript
interface DeathReason {
  reason: string;       // Короткое название
  description: string;  // 2–3 предложения
  severity: number;     // 1–10
  evidence: string;     // Подтверждающие данные
}

interface SuccessDriver {
  driver: string;       // Короткое название
  description: string;  // 2–3 предложения
  strength: number;     // 1–10
  evidence: string;     // Подтверждающие данные
}

interface CriticalReport {
  verdict: "HIGH_RISK" | "MEDIUM_RISK" | "LOW_RISK";
  overallScore: number;
  deathReasons: DeathReason[];
  successDrivers: SuccessDriver[];
  summary: string;
}
```
